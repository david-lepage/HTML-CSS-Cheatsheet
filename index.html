David's Cheatsheet for CSS Selectors and Specificity

What is the purpose?

The purpose of CSS Specificity is to ensure the correct style is applied to certain elements in the correct order. For example if two CSS rules were written targeting one element, the browser would need to know which CSS rule should be applied.

How its applied

Types of Specificity
Type of Rule Explanation
Last Rule If the two rules in CSS are identical then the latter will be applied
Specificity If one selector is more specific than the other then it will be applied
!important You can add this after any property to make it president over all other rules. It is important that this isnt of used within a page.

In what order is the Specificity?

Highest to Lowest Priority
Type Code Used in CSS
Important Coding !important
ID Selector [#] Hashtag Name
Class Selector [.] Dot Classname
Type Elements e.g. H1 or Pseudo Type Name

It is important to be aware of multiple classifications for example h1.even (all h1 with an class of even) or h1#time (all h1 with a ID of time) will be more specific than a class or ID on its own. So h1#time would be more specific than h1 but not more than #time or h1.time would be more specific than h1 but not more than .time
